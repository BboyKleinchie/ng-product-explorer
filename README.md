# Product Explorer

This is a web application for an internal tools team which allows users to browse a catalog, search/filter products, view product details, and save items to favorites.

This project follows the Angular styling guide which can be found [here](https://angular.dev/style-guide).

This project adheres to the Feature-based folder structure. see [blog](https://www.angular.courses/blog/angular-folder-structure-guide).

This project was generated using [Angular CLI](https://github.com/angular/angular-cli) version 21.1.1.

## Styling

This project uses the SCSS syntax which is the more preferred sassy syntax as it uses braces and semicolons similar to CSS.
We are using the mobile-first approach.

### Architecture

Using the 7-1 Pattern in this project as the SASS architecture to manage SASS files.

The architecture is customized a bit to suit the structure as made redundant by Angular, as follows:

1. Individual components, e.g. _app.scss_, will only style the component as generated by Angular. Commonly shared components, such as a button, dropdown, card, etc, will be set in the **/Components** SASS folder.

1. The **/Pages** SASS folder was made redundant by Angular and was removed from SASS folders
1. The **/Vendors** SASS folder was also removed because it was made redundant by Angular, as external stylesheets (and scripts) should be included in _angular.json_.
1. Component variables that will change depending on the theme, should use a css property variable that can be changed in the **/Themes**.

### Methodology

Using the BEM methodology, i.e. `Block__Element--Modifier` naming of classes.

_see:_ [BEM](http://getbem.com/) _for more details_

Using CSS Globals and CSS Utility Classes along with BEM.

_see:_ [Scalable CSS](https://css-tricks.com/building-a-scalable-css-architecture-with-bem-and-utility-classes/) _for more details_
